Mapa de Operação - Uardon

Versão 1 página para operação diária e explicação comercial.

1) Onde fica o código

  • GitHub: viniciusamorim514/uardon
  • Branch de produção: main
  • CRM: crm_vitoria_source/
  • Landing: web/

2) Onde publica

  • CRM/API em Railway (app.uardon.com.br)
  • Landing em Cloudflare Pages/Workers (uardon.com.br)
  • Fluxo: push no GitHub → deploy automático

3) Domínio e DNS

  • Gerenciamento DNS: Cloudflare
  • Domínio principal: uardon.com.br
  • Subdomínio CRM: app.uardon.com.br

4) Onde ver logs

  • Railway: Deploy Logs + HTTP Logs
  • CRM: /admin/auditoria-auth
  • Ver login, reset, ações admin e bloqueios

5) Onde ver e-mail

  • Envio transacional: Resend
  • Caixa operacional: Umbler Webmail
  • Operação: suporte@uardon.com.br
  • Remetente técnico: noreply@uardon.com.br

6) Responsáveis

  • Produto/comercial: Vitória Uardon
  • CRM e operação: Responsável CRM
  • Infra/deploy/DNS: Responsável técnico
  • Atendimento caixa suporte: Responsável suporte

7) Rotina diária (5 minutos)

health deploy auditoria auth e-mail operacional novos leads

8) Escalonamento rápido

  • Falha login/reset: auditoria auth + logs Railway + painel Resend.
  • Falha deploy: Deploy Logs no Railway + último commit.
  • Falha domínio: DNS da Cloudflare (zona uardon.com.br).
  • Falha e-mail: status Resend + caixa suporte na Umbler.

9) Mensagem comercial padrão

A Uardon opera com arquitetura separada para estabilidade: landing em Cloudflare e CRM/API em Railway com banco Postgres, deploy automatizado por GitHub, trilha de auditoria e monitoramento de autenticação.
